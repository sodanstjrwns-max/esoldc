# 🔍 이솔치과의원 검색엔진 색인(SEO 인덱싱) 가이드

> **목표:** 구글 · 네이버 · 빙이 `https://isoldc.kr`를 검색 결과에 노출시키도록 등록한다.
> **소요 시간:** 약 30분 (3개 사이트 합쳐서)
> **준비물:** 네이버 아이디, 구글 계정(Gmail), (선택) 마이크로소프트 계정

---

## ✅ 이미 끝나 있는 것 (개발자가 처리 완료)

| 항목 | 상태 | 설명 |
|------|------|------|
| `sitemap.xml` (사이트맵) | ✅ | 전체 페이지 목록 자동 생성 — https://isoldc.kr/sitemap.xml |
| `robots.txt` | ✅ | 검색봇 허용 규칙 — https://isoldc.kr/robots.txt |
| canonical / hreflang | ✅ | 중복 색인 방지, 한국어 명시 |
| OG / 트위터 카드 | ✅ | 카톡·SNS 공유 시 썸네일·제목 |
| 구조화 데이터(JSON-LD) | ✅ | 병원 정보를 검색엔진이 이해하도록 |
| AI 크롤러 허용 (llms.txt) | ✅ | ChatGPT·퍼플렉시티 등 답변 인용 허용 |
| **소유권 확인 메타태그 자리** | ✅ | 코드만 받아오면 한 줄로 활성화 (아래 참조) |
| **자동 색인 (IndexNow)** | ✅ | 글·비포애프터 발행 즉시 검색엔진에 자동 통보 ⭐ |

→ **기술 준비는 100% 완료.** 원장님은 "내 사이트가 맞다"고 증명하고 등록 버튼만 누르시면 됩니다.

---

## ⚡ 자동 색인 (IndexNow) — 컬럼/비포애프터 올리면 자동으로!

> **원장님 요청사항이 바로 이것!** "컬럼이나 비포애프터 업로드할 때 자동 색인"

### 어떻게 동작하나요?
1. 관리자에서 **블로그 글**이나 **비포&애프터**를 **"공개"로 발행**하는 순간
2. 시스템이 자동으로 검색엔진(빙·얀덱스·구글)에 **"새 글 올라왔어요, 지금 와서 보세요!"** 신호 전송
3. 사이트맵도 자동 갱신 → 평소보다 **훨씬 빠르게 색인**

→ **원장님이 따로 할 일 없음.** 그냥 평소처럼 발행 버튼만 누르면 끝. 🎉

### 수동 일괄 재요청 (필요할 때만)
- 관리자 대시보드 > **빠른 작업 > "검색엔진 색인 요청"** 버튼
- 전체 주요 페이지 + 발행된 모든 글/케이스를 한 번에 재요청
- 예: 사이트 대규모 개편 후, 또는 한 번에 여러 글을 정리했을 때

### ⚠️ 참고 (정직하게)
- IndexNow는 **빙·얀덱스가 공식 지원**, **구글도 신호를 받지만** 구글은 자체 크롤링 비중이 큼
- 그래서 **구글 서치콘솔 등록(아래 1번)도 꼭 같이** 해야 완성됩니다
- 색인 "요청"이지 "상위 노출 보장"은 아닙니다 — 콘텐츠가 쌓일수록 효과 ↑

## 📌 핵심 개념 (1분 이해)

검색엔진 등록은 딱 2단계입니다:

1. **"이 사이트 내 거 맞아요" 증명하기** → 각 웹마스터 도구가 *확인 코드*를 줍니다.
2. **그 코드를 사이트에 심기** → 개발자(저)에게 코드만 전달 → 한 줄 수정으로 끝.
3. (자동) **사이트맵 제출** → 검색엔진이 알아서 전체 페이지를 긁어갑니다.

# 1️⃣ 구글 (Google Search Console) — 가장 중요 ⭐

검색 트래픽의 70%+. 여기부터 하세요.

### Step 1. 접속
- https://search.google.com/search-console 접속 → Gmail 로그인

### Step 2. 속성 추가
- 왼쪽 상단 **"속성 추가"** 클릭
- 두 가지 선택지 중 **오른쪽 "URL 접두어"** 선택
- 입력칸에 `https://isoldc.kr` 입력 → **계속**

### Step 3. 소유권 확인 코드 받기
- 확인 방법 중 **"HTML 태그"** 펼치기
- 이렇게 생긴 코드가 나옵니다:
  ```html
  <meta name="google-site-verification" content="여기에긴문자열" />
  ```
- **`content="..."` 안의 긴 문자열만 복사** (예: `aBcD1234efGH...`)

### Step 4. 개발자에게 전달
> **"구글 확인 코드: aBcD1234efGH..."** 라고 채팅으로 보내주세요.
> → 제가 `clinic.ts`의 `siteVerification.google`에 넣고 재배포합니다 (1분).

### Step 5. 확인 버튼 클릭
- 재배포 완료 후, 구글 콘솔로 돌아가 **"확인"** 클릭 → ✅ 소유권 확인됨

### Step 6. 사이트맵 제출
- 왼쪽 메뉴 **색인 > Sitemaps**
- 입력칸에 `sitemap.xml` 입력 → **제출**
- "성공"이 뜨면 끝. (며칠 내 색인 시작)

# 2️⃣ 네이버 (서치어드바이저) — 국내 필수 ⭐

국내 검색·지도 노출의 핵심. 구글 다음으로 중요.

### Step 1. 접속
- https://searchadvisor.naver.com 접속 → 네이버 로그인
- 상단 **"웹마스터 도구"** 클릭

### Step 2. 사이트 등록
- 입력칸에 `https://isoldc.kr` 입력 → 추가(+) 버튼

### Step 3. 소유 확인 코드 받기
- **"HTML 태그"** 방식 선택
- 이런 코드가 나옵니다:
  ```html
  <meta name="naver-site-verification" content="여기에문자열" />
  ```
- **`content="..."` 안의 문자열만 복사**

### Step 4. 개발자에게 전달
> **"네이버 확인 코드: ..."** 라고 보내주세요. → 한 줄 수정 + 재배포.

### Step 5. 확인 + 사이트맵 제출
- 재배포 후 네이버에서 **"소유확인"** 클릭
- 확인되면: **요청 > 사이트맵 제출** → `https://isoldc.kr/sitemap.xml` 입력 → 확인
- 추가로 **요청 > RSS 제출**도 있으면 같이 (선택)

### 💡 네이버 보너스
- **요청 > 웹 페이지 수집**에서 `https://isoldc.kr` 직접 넣으면 수집 속도 ↑
- **네이버 플레이스(지도)** 등록은 별도입니다 → 아래 3번 참조

---

# 3️⃣ 빙 (Bing Webmaster Tools) — 보너스

점유율은 낮지만 **무료 + ChatGPT 검색이 빙 기반**이라 해두면 좋음.

### 쉬운 방법: 구글에서 가져오기 (강추)
- https://www.bing.com/webmasters 접속 → 마이크로소프트 계정 로그인
- **"Google Search Console에서 가져오기"** 클릭
- 구글 계정 연결 → 사이트 목록에서 `isoldc.kr` 선택 → **가져오기**
- → 끝! (사이트맵까지 자동으로 넘어옴, 별도 코드 불필요)

### 수동 방법 (구글 가져오기 안 될 때)
- "사이트 수동 추가" → `https://isoldc.kr` 입력
- **"HTML Meta Tag"** 방식 → `<meta name="msvalidate.01" content="...">` 의 content 복사
- 개발자에게 **"빙 확인 코드: ..."** 전달 → 재배포 → 확인

# 4️⃣ 네이버 플레이스(지도) — 별도 ⭐ 환자 유입 핵심

> ⚠️ 위 3개는 "홈페이지 검색 노출"이고, 이건 "지도/플레이스 노출"로 **별개**입니다.
> 남양주·마석 환자가 "마석 치과" 검색하면 뜨는 그 영역. **매우 중요.**

- **네이버 스마트플레이스**: https://smartplace.naver.com
- 사업자 인증 후 병원 정보(주소·진료시간·전화·사진·소개) 등록
- 등록되면 **플레이스 URL**이 생깁니다 (예: `https://map.naver.com/p/entry/place/12345678`)
- 그 URL을 저에게 주시면 → 홈페이지 "리뷰 안내 / 오시는 길" 버튼이 **플레이스로 직접 연결**됩니다 (현재는 검색 안내로 임시 동작 중)

## 🔧 개발자(저)가 코드 받으면 하는 일 (참고용)

`src/data/clinic.ts`의 아래 부분에 코드만 채우고 재배포:
```typescript
siteVerification: {
  google: '받은코드',   // ← 여기
  naver: '받은코드',    // ← 여기
  bing: '받은코드',     // ← 여기
},
```
→ 빌드 → 배포 → 끝. 빈 값은 자동으로 무시되므로 깨질 일 없음.
